Basic View Markup

Every configured route loads a partial (a piece of html) dynamically. These partials represents the pages of your mobile website or application.

The simplest markup looks like this:


<div ng-controller="MyCoolCntl">
    <div id="wrapper">
        <div id="scroller">
           ... the content goes here ...
        </div>
    </div>
</div>

MyCoolCntl is the example name of the angular controller. See Angular.js documentation for further informations.

The both div containers with id "wrapper" and "scroller" must be there to make the header and footer sections fix and the content area scrollable. This is possible by using the jScrollPane Library. An alternative might be iScroll4, but i've replaced it by jScrollPane. (Feel free to use iScroll4 instead if you prefer it. See ng-mobile.js to get it work).

Inner wrap to get some spacing

If you want to add some spacing between content and the left and the right display border you can wrap your page content in a wrapper:


<div class="inner-wrap">
... the content goes here ...
</div

Place this wrapper inside of the container with id "scroller"

This can be helpful in some cases (e.g. if you have a lot of text), but you can also use the CSS class inset to add some space around particular elements, like ul.listview, because the ul.listview uses 100% of screen width although inside a inner-wrap.

Header

Often the header holds the page name only. This is easy. Just set an title in the controllers:


$scope.setHeaderText('My header text');

You can override the default header inside of yout partials by placing an div#header with some content. Then you can add some other markup, e.g. some buttons:


<div id="header">
    <h1>{ {txtHeader} }</h1>
</div>

Back-Button

You can place a back-button in your header partial or inside your custom header container. The Button text will be automatically the title of the previous seen page. The button will be hidden, if no previous page exists:


<div id="header">
    <a href="{ {previousPage.hash} }" ng-show="previousPage.txtHeader" class="btn back">{ {previousPage.txtHeader} }</a>
    <h1>{ {txtHeader} }</h1>
</div>

Footer

The footer contains the navigation in most cases. By default in index.html loads the partial "partials/default_navigation.html" to footer.

You can override the footer for some pages, if you want. Just include a #footer in the partial:


<div id="footer">
    <ul>
        <li><a href="#start" class="icon home"><span>Start</span></a></li>
        <li><a href="#basics" class="icon pencil"><span>Basics</span></a></li>
    </ul>
</div>

Another way to override is to set the global "footer" element content in your controllers:


$scope.setPartial('footer', 'partials/other_footer.html');

Navigation

By default the navigation is placed in the footer of your mobile page, like seen above. This is a good idea for portrait orientation of a mobile device. But if one rotate it, it would be nice to have navigation at the left side instead.

This is done by CSS media queries in ng-mobile.sass. The element nav will be used to show a navigation at the left, while footer is hidden.

nav is placed in index.html even like header and footer. It loads the default partial "default_navigation.html" and can be overridden by a div with id "nav" in partials.

Note: Minimum width in landscape mode must be 600px, otherwise the footer remains.


<div id="footer">
    <ul>
        <li><a href="#start" class="icon home"><span>Start</span></a></li>
        <li><a href="#basics" class="icon pencil"><span>Basics</span></a></li>
    </ul>
</div>
<div id="nav">
    <ul>
        <li><a href="#start" class="icon home"><span>Start</span></a></li>
        <li><a href="#basics" class="icon pencil"><span>Basics</span></a></li>
    </ul>
</div>

Another way to override is to set the global "nav" element content in your controllers:


$scope.setPartial('navigation', 'partials/other_navigation.html');

Use the same Template for footer and nav

Perhaps it would be nicer to use the same template for both? Dead simple!


<div id="footer" ng-include="navInc"></div>
<div id="nav" ng-include="navInc"></div>

In your controller you have to fill the variable for "navInc" (you can rename it of course...):


$scope.navInc = 'partials/my_custom_navigation.html';

Another way to override is to set the global "nav" and "footer" element content in your controllers:


$scope.setPartial('footer', 'partials/other_navigation.html');
$scope.setPartial('navigation', 'partials/other_navigation.html');

In this case you should not have a #footer and #nav containers in your partial...

No left sided navigation please!

If you don't want to use left side navigation, it is neseccary you override the media query (see ng-mobile.sass) in your custom-theme.sass.
