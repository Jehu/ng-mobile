How to start

It's really easy to start, if you are an webdeveloper with some experience. Let's go!

  • You should have run a ready to start webserver (apache or so) with an working vhost for your project
  • Go to https://github.com/Jehu/ng-mobile/downloads and hit one of the download buttons.
  • Unpack the downloaded archive to your vhost httpdocs directory
  • Point your browser to the project.

Congrats! You see your ng-mobile boilerplate.

First: Read

ng-mobile is a collection of widespread technologies and tools - nothing more. It's possible, you know the most of them already. If not, now it's time to learn some new exciting things...

  • Angular.js (it's a must to know this)
  • SASS (enhance the work with CSS, highly recommended)
  • Compass (uses SASS and make it even better. highly recommended)
  • jQuery (you know it already, i hope)
  • iScroll4 (is uses by default in ng-mobile, you should know that it exists)
  • html5boilerplate (mobile) (is uses by default in ng-mobile, you should know that it exists)

Second: Write

Now you are ready to write your own mobile website or application.

Rename "app.dist.js" to "app.js" and change the index.html to load the new "app.js"

app.js

As you can see, there is a ngMobile.config() at the beginning. This is the point where you wire URL, Controller and the view together (Routing). Change this section to your needs.

There are some example controllers in app.js. You can rename them for your needs. But please think of write a routing rule for it also.

Partials

In the /partials directory you find some views. This is the place where you add all templates for your mobile pages. Again in the routing section (app.js) you have to use the right filenames.

The views "default_footer.html" and "default_header.html" are loaded from the MainCntl and are used for all views by default. You can overload other header/footers in your controllers or include a div container with id "header" and/or "footer" in your views to use them only in this particular view.

In some cases it may be necessary to load a new global header and/or footer. You can do this by use the setPartial() method in your controller:

$scope.setPartial('header', 'partials/other_header.html');
$scope.setPartial('footer', 'partials/other_footer.html');

In most cases it will be a good idea to use default_header.html and default_footer.html and change them for your needs.
